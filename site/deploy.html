Một secret. Deploy trọn gói.

Credential, runtime config, workers.dev và smoke test cùng nằm trong CLOUDFLARE_CONFIG_JSON. Field tùy chọn thiếu thì CI bỏ qua.

Mẫu đầy đủ

{"accountId":"...","apiToken":"...","apiKey":"...","allowedModels":"@cf/moondream/moondream3.1-9B-A2B","defaultModel":"@cf/moondream/moondream3.1-9B-A2B","maxImageBytes":"8388608","fetchTimeoutMs":"12000","testImageUrl":"https://public.example/test.png","workersSubdomain":"my-team"}

Dùng Global Key bằng cách thay apiToken bằng email và globalApiKey.

Field runtime

FieldTác dụng
apiKeySecret bảo vệ API Worker
allowedModelsModel allowlist, phân cách bằng dấu phẩy
defaultModelModel mặc định
maxImageBytesGiới hạn byte ảnh
fetchTimeoutMsTimeout tải ảnh
testImageUrlẢnh public an toàn cho smoke test
workersSubdomainPrefix account workers.dev, không kèm đuôi

Lấy credential

  1. Account ID: Account home → menu cuối dòng account → Copy account ID.
  2. Email: email chính xác của user sở hữu Global API Key trong My Profile.
  3. Global API Key: My Profile → API Tokens → API Keys → View.
  4. API Token: Create Token → Custom Token; cấp Workers Scripts Edit và quyền Workers AI cần thiết.
  5. apiKey: tự tạo bằng openssl rand -hex 32; đây không phải credential Cloudflare.

Không đưa key vào code, issue, log, ảnh chụp, tài liệu hoặc chat.

Thêm vào GitHub

  1. Mở Actions secrets.
  2. New repository secret, tên CLOUDFLARE_CONFIG_JSON.
  3. Dán JSON một dòng, nháy kép thẳng, không dấu phẩy cuối.
  4. Actions → Test, deploy Worker and publish docs → Run workflow.

CI tự làm gì?

  1. Chạy typecheck, script syntax check và unit tests.
  2. Tạo config runtime tạm, mask bí mật.
  3. Tùy chọn tạo account workers.dev subdomain.
  4. Deploy, bắt URL, bật route workers.dev.
  5. Set Worker API_KEY bằng secret store.
  6. Gọi /health và /v1/extract; thiếu rawText là fail.

Log smoke test chỉ hiện ok, model và request ID. GitHub Pages là tài liệu; API thật nằm ở URL workers.dev.
